# /save-memory — persist this session into the vault

Capture this session into `~/vault` while the conversation context is still
warm: write the raw transcript, then hand it to `/ingest-session` to distill
into decisions/notes — extraction always reads the persisted transcript file,
never this conversation's own (possibly already-compacted) memory of itself.
See the "Context Navigation (memory + graphify)" section of the global config
for how the graphs fit together.

Run from **any project directory** — this skill resolves the vault path itself;
the vault does not need to be the cwd.

The project's home in the vault is `~/vault/memory/projects/<project>/` with
`decisions/`, `notes/`, `sessions/` (graphed) and `chats/` (raw, excluded). The
map/index is generated by graphify from the graph — not hand-written here.

## Steps

1. **Resolve the project name and paths:**
   ```bash
   project="$(basename "$(git rev-parse --show-toplevel 2>/dev/null || pwd)")"
   date="$(date +%F)"
   proj_dir=~/vault/memory/projects/"$project"
   mkdir -p "$proj_dir"/decisions "$proj_dir"/notes "$proj_dir"/chats "$proj_dir"/sessions
   # Per-project ignore, REQUIRED at the scan root: graphify's ignore walk stops
   # at the nearest VCS root, so unless ~/vault is a git repo the vault-level
   # .graphifyignore is never consulted when scanning $proj_dir — without this
   # file the chats/ transcripts would leak into the memory graph.
   [ -f "$proj_dir"/.graphifyignore ] || printf 'chats/\ndoc/\n' > "$proj_dir"/.graphifyignore
   ```

2. **Load the vault's write-rules — do NOT hardcode them here.** Read
   `~/vault/CLAUDE.md` and follow its Zettelkasten rules verbatim (mandatory YAML
   frontmatter, kebab-case filenames, wikilinks not markdown links, ≥2 wikilinks
   per permanent note, one idea per note). `~/vault/CLAUDE.md` is the single
   source of truth for *how* the vault is written; this skill decides *what* to
   write and *where*.

3. **Dump the raw session transcript** into `$proj_dir/chats/` as
   `<date>-<slug>.md` (`<slug>` = a few kebab-case words). Locate the current
   session's JSONL — the newest file under
   `~/.claude/projects/<cwd-slug>/*.jsonl` (cwd-slug = the cwd with `/`→`-`) — and
   render it to markdown (reuse `claude-extract` or
   `~/claude-code-memory-setup/scripts/claude_to_obsidian.py`). This is the raw
   record; it is excluded from the graph by `.graphifyignore`.

4. **Distill it into atomic notes — do NOT hand-author them here.** Invoke
   `/ingest-session <path-to-the-transcript-just-written>` (see
   `skills/ingest-session/SKILL.md`). It reads the transcript file itself
   (ground truth, not this conversation's own memory of itself — which may
   already be a lossy `/compact` summary), extracts decisions/notes, dedups
   against existing ones, and writes the mirrored `sessions/summary-*.md`
   processed marker. Do NOT hand-maintain an index file — the graph generates
   the map (see step 6).

5. **Flag promotion candidates.** The merged graph (step 6) already bridges these
   vault notes to code, but only *locally* (vault nodes, machine-local). If a
   captured decision is *durable AND repo-specific* (an ADR a teammate cloning the
   repo would want), note in its frontmatter/body that it's a candidate to
   **graduate to the portable repo L1** — move it into a repo doc, commit it, and
   run `graphify link-docs` against the repo graph so it survives clone/CI. Don't
   promote fluid, personal, or cross-project notes; those stay in the vault.

6. **Sync the graph pipeline: refresh the two inputs, re-link, export briefings.**
   The query surface is the *merged* graph
   (`$proj_dir/graphify-out/merged.json` = repo graph ⊕ memory graph ⊕ bridges).
   Refresh both inputs, compose, then export the cached briefings `/load-memory`
   reads (no hand-written INDEX):
   ```bash
   repo_graph="$(git rev-parse --show-toplevel 2>/dev/null)/graphify-out/graph.json"
   mem_graph="$proj_dir"/graphify-out/graph.json
   merged="$proj_dir"/graphify-out/merged.json
   # 1. inputs
   graphify update . 2>/dev/null || true                     # repo graph (source)
   # memory graph (notes only) — ALWAYS `extract --doc-only`, never `update`:
   # `update` is unconditionally AST-only ("re-extract code files, no LLM
   # needed") and succeeds even on a notes dir, silently downgrading the
   # semantic doc graph to a structural one (one node per file/heading) with
   # no error to catch — found the hard way, see graphify note
   # `notes/update-vs-extract-doc-only.md`.
   graphify extract "$proj_dir" --doc-only 2>/dev/null || true
   # 2. link the two graphs -> merged query surface (machine-local; inputs read-only)
   graphify link-docs "$proj_dir" \
       --code-graph "$repo_graph" --doc-graph "$mem_graph" \
       --match-code --link-code --out "$merged" 2>/dev/null || true
   # 3. briefings (pre-defined queries, cached as markdown)
   graphify export wiki --graph "$merged" 2>/dev/null || \
     graphify export wiki --graph "$mem_graph" 2>/dev/null || true
   #   → $proj_dir/graphify-out/wiki/index.md   = cached briefing
   ```
   The merged graph is **machine-local** (it contains vault nodes) — do NOT write
   these bridges into the committed repo graph (`--out` keeps them in the vault;
   `link-docs` never mutates its `--code-graph`/`--doc-graph` inputs, and stamps
   `merged.json` with both input hashes under `graph.link_meta`). **Do not**
   rebuild the **global tier** here — it's *composed* from the memory graphs on
   demand, not per save:
   ```bash
   # occasional cross-project view (not every save). merge-graphs namespaces every
   # node id as <project>::<id> and tags provenance (node attr `repo`) — right for
   # the cross-project tier, never for the per-project merged.json (prefixed ids
   # would break the manifest-validated bridges). Feed it the memory graph.jsons
   # only, NEVER the merged.jsons (code would flood the code-free tier):
   # graphify merge-graphs ~/vault/memory/projects/*/graphify-out/graph.json \
   #     ~/vault/memory/global/graphify-out/graph.json \
   #     --out ~/vault/memory/graphify-out/graph.json
   # graphify cluster-only ~/vault/memory   # re-cluster the union — the inputs'
   #                                        # community ids collide across projects
   # graphify export wiki --graph ~/vault/memory/graphify-out/graph.json
   #     → ~/vault/memory/graphify-out/wiki/index.md   = global index
   ```

7. **Commit the vault** if it is a git repo:
   ```bash
   git -C ~/vault add -A && git -C ~/vault commit -m "save: $project $date" && git -C ~/vault push 2>/dev/null || true
   ```
   (Follow the global git rule: no Claude/Anthropic co-author trailer.)

## Notes
- Don't delete or overwrite existing notes without asking (vault rule).
- Keep notes factual: if something was tried and failed, say so.
- Distilled notes are graphed; the transcript and everything outside
  `~/vault/memory/` are not.
