# /save-memory — persist this session into the vault (Layer 2)

Capture the session's *judgment* (decisions, why-X-over-Y, gotchas, what's left)
into `~/vault` while the conversation context is still warm. This is the
write-back step of the memory workflow — see the "Context Navigation (Graphify)"
section of the global config for how the layers fit together.

Run from **any project directory** — this skill resolves the vault path itself;
the vault does not need to be the cwd.

The project's home in the vault is `~/vault/memory/projects/<project>/` with
`decisions/`, `notes/` (graphed) and `chats/` (raw, excluded). The map/index is
generated by graphify from the graph — not hand-written here.

## Steps

1. **Resolve the project name and paths:**
   ```bash
   project="$(basename "$(git rev-parse --show-toplevel 2>/dev/null || pwd)")"
   date="$(date +%F)"
   proj_dir=~/vault/memory/projects/"$project"
   mkdir -p "$proj_dir"/decisions "$proj_dir"/notes "$proj_dir"/chats
   ```

2. **Load the vault's write-rules — do NOT hardcode them here.** Read
   `~/vault/CLAUDE.md` and follow its Zettelkasten rules verbatim (mandatory YAML
   frontmatter, kebab-case filenames, wikilinks not markdown links, ≥2 wikilinks
   per permanent note, one idea per note). `~/vault/CLAUDE.md` is the single
   source of truth for *how* the vault is written; this skill decides *what* to
   write and *where*.

3. **Capture durable judgment as atomic notes** — the graphed layer. Write one
   idea per note into `$proj_dir/decisions/` (ADRs, why-X-over-Y, rejected
   alternatives) and `$proj_dir/notes/` (permanent/concept notes), frontmatter +
   ≥2 wikilinks per the rules. Only capture what graphify could NOT regenerate
   from the repo (decisions, tradeoffs, gotchas) — never caller lists, signatures,
   or anything mechanically derivable. Link related notes to each other and, for
   decisions that replace earlier ones, add a `supersedes` wikilink — that
   *is* the "what's current" narrative; do NOT hand-maintain an index file (the
   graph generates the map — see step 6).

4. **Dump the raw session transcript** into `$proj_dir/chats/` as
   `<date>-<slug>.md` (`<slug>` = a few kebab-case words). Locate the current
   session's JSONL — the newest file under
   `~/.claude/projects/<cwd-slug>/*.jsonl` (cwd-slug = the cwd with `/`→`-`) — and
   render it to markdown (reuse `claude-extract` or
   `~/claude-code-memory-setup/scripts/claude_to_obsidian.py`). This is the raw
   record; it is excluded from the graph by `.graphifyignore`.

5. **Flag promotion candidates.** If a captured decision is *durable AND
   repo-specific* (an ADR a teammate cloning the repo would want), note in its
   frontmatter/body that it's a candidate to graduate to the repo's L1 — move it
   into a repo doc and run `graphify link-docs` so it bridges to the code. Don't
   promote fluid, personal, or cross-project notes; those stay in the vault.

6. **Refresh the project graph + its index.** Each project owns its own graph at
   `$proj_dir/graphify-out/`; update it (create on first run) and render the
   project index — this *is* the map `/load-memory` queries, no hand-written INDEX:
   ```bash
   graphify update "$proj_dir" 2>/dev/null || graphify "$proj_dir" 2>/dev/null || true
   graphify export wiki --graph "$proj_dir"/graphify-out/graph.json 2>/dev/null || true
   #   → $proj_dir/graphify-out/wiki/index.md   = per-project index
   ```
   **Do not** rebuild the vault-wide roll-up here — it's composed from the project
   graphs on demand/nightly (cheap merge, no re-extraction), not per save:
   ```bash
   # occasional, cross-project view (not every save):
   # graphify merge-graphs ~/vault/memory/projects/*/graphify-out/graph.json \
   #     --out ~/vault/memory/graphify-out/graph.json
   # graphify export wiki --graph ~/vault/memory/graphify-out/graph.json
   #     → ~/vault/memory/graphify-out/wiki/index.md   = vault index
   ```

7. **Commit the vault** if it is a git repo:
   ```bash
   git -C ~/vault add -A && git -C ~/vault commit -m "save: $project $date" && git -C ~/vault push 2>/dev/null || true
   ```
   (Follow the global git rule: no Claude/Anthropic co-author trailer.)

## Notes
- Don't delete or overwrite existing notes without asking (vault rule).
- Keep notes factual: if something was tried and failed, say so.
- Distilled notes are graphed; the transcript and everything outside
  `~/vault/memory/` are not.
